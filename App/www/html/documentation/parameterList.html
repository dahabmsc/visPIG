visPIG - Visual Plotting Interface for Genetics

Below are all the parameters that can be specified directly via visPIG. The parameter name is the name of the variable that you need to specify either on the command line or the parameter file, the short description corresponds to the description in the visPIG web application panels and the long description gives further explanations what each parameter does. The type & sensible values column lists what type of input is expected and what are sensible / possible value.

-------------------- data files --------------------
regions.file region file the full or relative path (incl. the filename) of the input file specifying the regions to be plotted character
parameters.file parameter file the full or relative path (incl. the filename) of the input file containing the visPIG parameters character
annot.files multi-state annotation file(s) (annot) the full or relative path (incl. the filename(s)) of the input file(s) containing the multi-state annotation data character; NB for multiple files, separate the different paths by commas (e.g. path1/file1,path2/file2,path3/file3 etc)
assoc.file association file (assoc) the full or relative path (incl. the filename) of the input file containing the association data character
dix.file directionality index file (dix) the full or relative path (incl. the filename) of the input file containing the directionality index data character
gerp.files GERP file (gerp) the full or relative path (incl. the filename(s)) of the input files containing the GERP conservation data character; separate multiple filenames by commas; NB multiple files do not result in multiple GERP tracks, but will simply be concatenated
heat.file heatmap file (heat) the full or relative path (incl. the filename) of the input file containing the heatmap data character
hic.files interaction file(s) (hic) the full or relative path (incl. the filename(s)) of the input file(s) containing the interaction data character; separate multiple filenames by commas
int.files intensity file(s) (int) the full or relative path (incl. the filename(s)) of the input file(s) containing the intensity data character; separate multiple filenames by commas
phast.files phastCons file(s) (phast) the full or relative path (incl. the filename(s)) of the input file(s) containing the phastCons conservation data character; separate multiple filenames by commas; NB multiple files do not result in multiple phastCons tracks, but will simply be concatenated
snap.data.files SNAP data file (snap) the full or relative path (incl. the filename(s)) of the input file(s) containing the SNAP association data character; separate multiple filenames by commas
snap.rate.files SNAP rate file (snap) the full or relative path (incl. the filename(s)) of the input file(s) containing the SNAP recombination rate data character; separate multiple filenames by commas
feat.files feature file (feat) the full or relative path (incl. the filename(s)) of the input file(s) containing the feature annotation data character; separate multiple filenames by commmas
lines.file vertical line file the full or relative path (incl. the filename(s)) of the input file(s) containing the coordinates for vertical lines to overplot on all tracks character
gene.file gene file (gene) the full or relative path (incl. the filename(s)) of the input file(s) containing the gene annotation data character
chrsizes.file chromosome sizes files the full or relative path (incl. the filename(s)) of the input file(s) containing the lengths (in base-pairs) of chromosomes; this will be used to computed genome-wide coordinates if the regions to be plotted lie on more than one chromosome character
track.order.top.down track order (top-down); leave blank for default order; you might want to know what's what specifies the order of tracks on the resulting plot; order is top-down; track codes are given in the visPIG menu in brackets behind the filename parameters and here; leave blank for default order; multiple tracks of one type can be either plotted as a block (e.g. int) or the order of individual instances of said types can be specified by appending the number of each instance behind the type identifier (e.g. int1,int2,int3) character; comma-separated list of track identifier codes
-------------------- general graphics --------------------
width.plot plot width (inches) defines the width of the png/pdf image in inches numeric; >0
height.plot plot height (inches) defines the height of the png/pdf image in inches; it is recomended not to set this parameter and let visPIG figure out (from the track specific parameters and data files) what the optimal height is; if you still decide to fix the plot height, make sure it is large enough or you can end up with a 'figure margins too large' error numeric; >0
left.margin left margin (inches) defines the size of the left figure margin in inches; increase if some of your y-axis labels do not fully print onto the figure numeric;
left.margin + right.margin < width.plot
right.margin right margin (inches) defines the size of the right figure margin in inches numeric;
left.margin + right.margin < width.plot
title.top.margin top margin (for title track; inches) defines the size of top margin (in inches) used on the title track only numeric; NB the sum of title.top.margin and sep.margin needs to be less than the size of the plot region reserved for the title track; if unspecified the value for sep.margin gets used
top.margin top margin (for axis track; inches) defines the size of top margin (in inches) used on the axis track only numeric; NB the sum of bottom.margin and top.margin needs to be less than the size of the plot region reserved for the axis track
bottom.margin bottom margin (for axis track; inches) defines the size of the bottom margin (in inches) used on the axis track only; NB the axis itself gets printed into the margin, so be sure to leave some bottom margin numeric; NB the sum of bottom.margin and top.margin needs to be less than the size of the plot region reserved for the axis track
sep.margin margin between tracks (inches) defines half the size (in inches) of the margin used between plots; an amount of sep.margin inches at the top and bottom of each track gets reserved for this margin; so the actual space between two tracks in 2*sep.margin numeric; NB the size of the plot region reserved for each track (except axis and title tracks which have their own top and bottom margins) needs to be larger than 2*sep.margin
ylab.las y-axis label orientation specifies the orientation of the y-axis labels; same values as for the las graphical parameter in R; since here it is applied only to the vertical axis: 0 & 3 = vertical; 1 & 2 = horizontal integer in {0,1,2,3}
letter.cex track letter font size character expansion factor used for the track letters; this is relative to the default, so values larger than 1 increase the font relative to the default; values less than 1, shrink it numeric; >0
xlab x-axis label specifies the x-axis label text character; NB on the web form specifying '*' will use a default label; if you save the parameter file this will be overwritten as NA and so no default will be used when that parameter file is loaded later on
xlab.cex x-axis label text size character expansion factor used for the x-axis label text; this is relative to the default, so values larger than 1 increase the font relative to the default; values less than 1, shrink it numeric; >0
axis.relplotheight x-axis relative track height every track (even the axis and title 'tracks') has an associated relative plot height; essentially if all tracks were to be plotted equally they will have the same relative plot height; NB this does not mean that they will take up the same amount of space on the resulting plot; tracks such as the heat map, the interaction (if not collapsed) and the gene tracks will take up more or less space depending on how wide a region is plotted, how many genes or interactions there are within the plotted regions etc.; values larger than 1 will increase the track's size relative to other plots, values less than 1 decrease it numeric, >0
axis.line x-axis margin inset specifies the number of lines into the margin the x-axis gets drawn; NB this can be negative and does not need to be an integer numeric; best to try first with value >=0, but negative non-integer values are allowed
xlab.line x-axis label margin inset specifies the number of lines into the margin the x-axis labels get printed; NB this can be negative and does not need to be an integer numeric; best to try first with value >=0, but negative non-integer values are allowed
axis.range.line x-axis range labels margin inset specifies the number of lines into the margin the x-axis range labels get printed; NB this can be negative and does not need to be an integer numeric; best to try first with value >=0, but negative non-integer values are allowed
axis.tickmarks.line x-axis tickmark labels margin inset specifies the number of lines into the margin the x-axis tickmark labels get printed; NB this can be negative and does not need to be an integer numeric; best to try first with value >=0, but negative non-integer values are allowed
axis.addlines over-impose vertical lines on the x-axis? specifies whether the vertical lines specified in the vertical line file gets plotted on the axis track also or not logical; TRUE/FALSE
x.lwd x-axis line width specifies the width of x-axis line numeric; >=0
x.axis.format x-axis units specifies the units of the x-axis (base pairs, kilo base pairs or mega base pairs) character; one of bp, Kb, Mb
x.labels.show.unit show the units on the x-axis labels? specifies whether the units should be displayed on the x-axis labels logical; TRUE/FALSE
axis.tcl length & direction of axis tickmarks specifies both the direction (sign) and length (numerical value) of the axis tickmarks; positive values mean the tickmarks extend above the axis line whereas negative values mean they extend below the axis line; you can specify multiple, commas-separated values -- this is useful for having tickmarks protruding on both sides of the axis line (e.g. -0.5,0.5) numeric; between -0.5 and 0.5
axis.lend line end style for the axis specifies the darwing style for the ends of the axis lines (incl. tickmarks) integer; 0 = round line caps, 1 = butt line caps, 2 = square line caps
axis.labels.slanted slant the axis labels? specifies whether the x-axis labels should be slanted or not logical; TRUE/FALSE
axis.labels.angle degrees of slanting (anti-clockwise) if axis.labels.slanted==TRUE, then this specifies the anti-clockwise degree of slanting numeric; between 0 and 360 (but any number, even negative ones, will work)
gap_width gap width (for gaps on x-axis) specifies the width of the gaps on the x-axis (if any) as a proportion of the total range plotted on the figure; e.g. if a plot consists of a region of 1.25Mb and one region of 0.75Mb and gap_width=0.1, then the gap between the two regions will look to be 0.1*(1.25+0.75)=0.2Mb wide numeric, >=0
gaps.sizes.offset.x gap label inset specifies the number of lines into the margin the labels indicating the gap sizes get drawn; NB this can be negative and does not need to be an integer numeric
title.relplotheight relative height for title track every track (even the axis and title 'tracks') has an associated relative plot height; essentially if all tracks were to be plotted equally they will have the same relative plot height; NB this does not mean that they will take up the same amount of space on the resulting plot; tracks such as the heat map, the interaction (if not collapsed) and the gene tracks will take up more or less space depending on how wide a region is plotted, how many genes or interactions there are within the plotted regions etc.; values larger than 1 will increase the track's size relative to other plots, values less than 1 decrease it numeric; >0
title.letter track letter for title track individual tracks can be labelled by letters / numbers (e.g. for easy reference in a caption); this specifies the letter / number for the title track character; single characters or letters would make most sense, but you can enter whole words
title title the text that should be printed as title for the plot character
title.cex.txt title font size character expansion factor used for the title; this is relative to the default, so values larger than 1 increase the font relative to the default; values less than 1, shrink it numeric; >=0
title.pos title position specifies the position of the title on the plot; if unspecified, the title is centered along the x-axis; but you can specify a chromosome and a position (in bp) on that chromosome to print the title at that position numeric; between 0 and the maximum position of the chromosome specified by title.chr
title.chr title chromosome specifies the chromosome for the position of the title on the plot; if unspecified, the title is centered along the x-axis; but you can specify a chromosome and a position (in bp) on that chromosome to print the title at that position integer; between 1 and the number of the last chromosome (max_chr)
title.showgaps indicate gap boundaries on title track? gap boundaries are indicated by dotted lines; this specifies whether these lines should be also over-imposed on the title track logical; TRUE/FALSE
buffer.relplotheight relative height for empty or buffer tracks every track (even the axis and title 'tracks') has an associated relative plot height; essentially if all tracks were to be plotted equally they will have the same relative plot height; NB this does not mean that they will take up the same amount of space on the resulting plot; tracks such as the heat map, the interaction (if not collapsed) and the gene tracks will take up more or less space depending on how wide a region is plotted, how many genes or interactions there are within the plotted regions etc.; values larger than 1 will increase the track's size relative to other plots, values less than 1 decrease it numeric; >0

legend.height height(s) of the legends specifies the height of the legend as a proportion of the plot. you can specify it in inches (if the values do not sum to 1; i.e. are not proportions, visPIG will compute proportions based on the numbers provided; i.e. height_prop_i=height_i/sum_i(height_i)). you can either specify one value for the whole legend, or a value for each row of the legend. numeric; >0

snap.snp2.file data file for SNP #2 the full or relative path (incl. the filename) of the input file containing the SNAP association data for the second reference SNP; since the data file contains information (R-squared, distance) that relate to one specific SNP, if a second reference SNP is to be plotted on the graph, a second input file has to be specified character
snap.snp2.snp reference SNP #2 specifies the second reference SNP; this SNP will be highlighted on the SNAP track and its name printed besides it character (SNP name)
snap.snp2.offset.x SNP label x-offset (in plot units) for the second reference SNP specifies the amount of space the second reference SNP label should be shifted to the right on the plot; this is in the plot units, i.e. base-pairs; can be negative numeric
snap.snp2.offset.y SNP label y-offset (in plot units) for the second reference SNP specifies the amount of space the second reference SNP label should be shifted to the top of the plot; this is in the plot units, i.e. on the same scale as the negative log of the association p-values; can be negative numeric
-------------------- vertical lines --------------------
lines.lwd line width(s) sets the width used to draw the lines specified in the lines.file input file; comma-separated list of values, one for each line numeric; >=0
lines.lty line type(s) sets the type of line used to draw the lines specified in the lines.file input file; comma-separated list of values, one for each line; 0=blank, 1=solid (default), 2=dashed, 3=dotted,4=dotdash, 5=longdash, 6=twodash integer; between 0 and 6
lines.col line colour(s) sets the colour used to draw the lines specified in the lines.file input file; comma-separated list of colour names, one for each line character (colour name(s))
lines.nonzoom show vertical lines on the main plot? determines whether the vertical lines specified in the vertical lines file should be drawn onto the main, non-zoomed, part of the plot logical; TRUE/FALSE
lines.zoom show vertical lines on the zoomed region? determines whether the vertical lines specified in the vertical lines file should be drawn onto the zoomed part of the plot logical; TRUE/FALSE
