visPIG - Visual Plotting Interface for Genetics

General notes.

Please click on the respective links below to see examples of the various input formats.
Your input files should have the same number of columns and mimic the way information is summarised (e.g. 'chr8' rather than '8' etc.) and use the same field delimiters.
The gene file should be in standard UCSC format and the SNAP files (data and rate) should be in standard format as required for the Broad Institute's SNAP plots.
All files (except SNAP and chromosome sizes files which do not require this) should be tab separated not simply whitespace separated.

Example files.

region file
1 row per region to be plotted; 3 columns: chromosome (chr), start position (SP), end position (EP); tab delimited; no header row
parameter file
1 row per region to be plotted parameter; 1 column only; row format: parameter_name=parameter_value; if starting a new parameter file, please always download either a default parameter file or one with your parameters by clicking the 'save the parameters' button on the main visPIG page; after a visPIG update you may have to add new parameters to your existing parameter files (simply download a default parameter file and check which parameters are missing from your exisiting file)
multi-state annotation file
1 row per contiguous same-state block along the genome; the format is the output bed format of ChromHMM: the relevant columns are the first four coloumns: chr, SP, EP, state, and the ninth column: colour (in rgb coordinates); tab delimited; no header row
association file
1 row per SNP / position; 3 columns: chr, position (pos), p-value (but could be any other intensity); tab-delimited; no header row
directionality index file
1 row per block of positions sharing one directionality index value; at least 4 columns: chr, SP, EP, dix_value1, dix_value2, ...; tab-delimited; no header row
NB if multiple directionality indices are to be plotted, the file needs to contain at least as many columns as the maximum column number specified in the 'column numbers' fields in the 'Directionality Index Track' panel.
GERP / phastCons file with blocks
1 row per block of positions sharing GERP / phastCons value; 4 columns: chr, SP, EP, value; tab-delimited; no header row
GERP / phastCons file without blocks
1 row per position; 3 columns: chr, pos, value; tab-delimited; no header row
GERP / phastCons filter file
1 row per positions, 2 columns: chr, pos; tab-delimited; no header row
NB only used when the GERP / phastCons data is specified without blocks
heatmap file (if type [heat.type] is set to "pos-pos")
1 row per cell of the heat map; 5 columns: chr1, pos1, chr2, pos2, value; tab-delimited; no header row
NB do not forget to specify the 'bin size' parameter on the 'Heatmap Track' panel if your heatmap has bins wider than 1bp.
heatmap file (if type [heat.type] is set to "general")
1 row per cell of the heat map; 6 columns: chr, SP, EP, y1, y2, value; tab-delimited; no header row
interaction file (if plotting style [hic.style] is set to "bins" or "stacked")
1 row per interacting bin; 5 columns: chr, SP, EP, reference status (0 = reference bin, 1 = non-reference), group (integers identifying the group the bin belongs to); tab-delimited; no header row
interaction file (if plotting style [hic.style] is set to "arches" or "flatarches")
1 row per interaction / pair of bins; 7 columns: chr1, SP1, EP1, chr2, SP2, EP2, p-value (or other numeric value giving interaction strength); tab-delimited; no header row
intensity file
1 row per block of positions sharing one intensity value; 4 columns: chr, SP, EP, value; tab-delimited; no header row
SNAP data file
1 row per SNP; standard SNAP data file format, 9 columns: SNP, proxy, distance, RSquared, chr, pos, p-value, pos, SNP type; tab- or space-delimited; has a header row
SNAP rate file
1 row per position; standard SNAP rate file format, though can have an optional third column, 2 or 3 coloumns: pos, recombination rate, chr (optional); tab- or space-delimited; has a header row
NB if the chromosome column is missing, you have to specify it in the 'SNAP Track' panel
feature file
1 row per feature to be annotated; 4 columns: chr, SP, EP, label; tab-delimited; no header row
vertical line file
1 row per vertical line; 2 columns: chr, pos; tab-delimited; no header
gene file
1 row per gene; standard UCSC bed format: 10 columns: chr, strand, transcription SP, transcription EP, coding SP, coding EP, exon count, exon SPs, exon EPs, gene name; tab-delimited; has a header row
chromosome sizes file
1 row only; as many columns as chromosomes, each giving the length of that chromosome in base-pairs
NB you can safely round the sizes up or use very large sizes to make sure there is no overlap; the chromosome file provided has chromosomes 1 to 22, then X, Y, MT
multi-state annotation legend file
1 row per state, 2 columns; the first column gives the state names, the second the corresponding colour, either as rgb values or colour name
